# UI/UX Designer — Skills Profile for Resume Tailor
**Role:** UI/UX Designer  
**Project:** Resume Tailor - ATS Resume Optimization Platform  
**Date:** May 2, 2026

---

## Role Overview

The UI/UX Designer will create intuitive, accessible, and visually compelling user interfaces for Resume Tailor, a web-based SaaS platform that helps job seekers optimize their resumes for Applicant Tracking Systems (ATS). The designer will translate user needs and product requirements into wireframes, mockups, interactive prototypes, and a cohesive design system.

---

## Core Competencies Required

### 1. **User Experience (UX) Design**

#### Research & Discovery
- **User Research**: Conduct user interviews, surveys, and usability studies
- **Persona Development**: Create detailed user personas (job seekers, freshers, IT professionals, mid-level professionals)
- **Journey Mapping**: Map user flows from resume upload → JD input → analysis → results → export
- **Competitive Analysis**: Analyze competitor products (Rezi, Jobscan, Resume Worded, TopResume)
- **Information Architecture**: Structure content and navigation for optimal user comprehension

#### UX Strategy
- **User Story Translation**: Convert product requirements into design solutions
- **Interaction Design**: Define user interactions, transitions, and micro-interactions
- **Usability Testing**: Plan and conduct usability tests, gather feedback, iterate
- **Accessibility Standards**: Ensure WCAG 2.1 AA compliance (screen readers, keyboard navigation, color contrast)
- **Responsive Design Strategy**: Plan for desktop, tablet, and mobile experiences

### 2. **Visual Design (UI)**

#### Design Fundamentals
- **Typography**: Select and pair fonts for readability and brand identity
- **Color Theory**: Create accessible color palettes with proper contrast ratios (4.5:1 minimum)
- **Layout & Composition**: Design balanced, scannable layouts with clear visual hierarchy
- **Spacing & Grids**: Use consistent spacing systems (8px grid, 4px baseline)
- **Iconography**: Design or select icon sets that are clear and consistent

#### Visual Identity
- **Brand Guidelines**: Create or follow brand identity (logo usage, color palette, voice)
- **Design Systems**: Build reusable component libraries (buttons, forms, cards, modals)
- **Style Guides**: Document design patterns, component states, and usage guidelines
- **Asset Creation**: Create logos, illustrations, graphics, and marketing assets

### 3. **Prototyping & Wireframing**

#### Low-Fidelity Design
- **Sketching**: Rapid ideation through hand sketches or digital sketches
- **Wireframing**: Create low-fidelity wireframes for all key screens (Balsamiq, Figma, Sketch)
- **Paper Prototyping**: Quick validation through paper prototypes

#### High-Fidelity Design
- **Mockups**: Create pixel-perfect, high-fidelity mockups with final visuals
- **Interactive Prototypes**: Build clickable prototypes to simulate user flows
- **Animation & Transitions**: Define UI animations, loading states, and transitions
- **Component States**: Design hover, active, disabled, error, and loading states

### 4. **Design Tools Proficiency**

#### Essential Tools (Must Have)
- **Figma** (Primary): Wire framing, mockups, prototyping, design systems, collaboration
- **Adobe XD**: Alternative prototyping and design tool
- **Sketch**: macOS-based design tool (if team uses Sketch)

#### Supporting Tools
- **Adobe Creative Suite**:
  - Photoshop: Image editing, photo manipulation
  - Illustrator: Vector graphics, icon design, logos
  - After Effects (Optional): Animation and motion graphics
- **InVision**: Prototyping and collaboration
- **Framer** (Optional): Advanced interactive prototypes
- **Principle / ProtoPie** (Optional): Animation and interaction design

#### Collaboration & Handoff
- **Zeplin / Avocode**: Design handoff and specifications for developers
- **Abstract**: Version control for design files
- **Miro / FigJam**: Collaborative whiteboarding and brainstorming
- **Notion / Confluence**: Documentation and knowledge sharing

## Project-Specific Requirements for Resume Tailor

### 1. **Key Screens to Design**

#### Landing Page
- Value proposition messaging
- "Upload Resume" and "Upload Job Description" CTAs
- Trust indicators (security, testimonials, sample results)
- Clean, welcoming hero section

#### Resume Upload Screen
- Drag-and-drop file upload area
- File type validation (PDF, DOCX)
- Progress indicators (uploading → parsing → complete)
- Error states (invalid file, file too large, parsing failed)
- Success confirmation with preview of extracted data

#### Job Description (JD) Upload Screen
- File upload OR text paste area
- URL paste option (pull JD from job boards)
- Similar states: progress, error, success
- Preview of extracted keywords and requirements

#### Results Dashboard
- **ATS Score**: Visual score display (gauge, percentage, grade)
- **Gap Analysis**: Missing skills, keyword gaps, formatting issues
- **Side-by-Side Comparison**: Resume vs JD keywords
- **Suggestions Panel**: Actionable recommendations for improvement
- **Export Options**: Download optimized resume (PDF/DOCX)

#### Feedback & Settings
- User feedback form (rating, comments)
- Account settings (if authentication added later)
- History of previous analyses

### 2. **Design Considerations for Resume Tailor**

#### User Trust & Credibility
- Professional, credible design (not flashy or gamified)
- Clear data privacy messaging
- Show "processing" states to build confidence in AI analysis
- Use real resume examples in onboarding/demos

#### ATS-Specific Design Challenges
- Visualize complex scoring algorithms in simple terms
- Make keyword matching intuitive (highlight matching/missing keywords)
- Design resume comparison views (before/after)
- Show resume formatting issues visually

#### Responsive Design
- **Desktop First**: Primary use case (users will edit resumes on desktop)
- **Mobile Friendly**: View results, quick checks on mobile
- **Tablet Support**: Touch-friendly upload and review

#### Accessibility
- Screen reader compatibility for visually impaired users
- Keyboard navigation for all interactive elements
- High contrast mode support
- Large, readable fonts (minimum 16px body text)

## Success Metrics for UI/UX Designer

### Design Quality
- **User Satisfaction**: High usability test scores (>80% task completion)
- **Design Consistency**: All screens follow design system
- **Accessibility**: WCAG 2.1 AA compliance (verified by audit)

### Business Impact
- **Conversion Rate**: Increase in resume upload → analysis completion
- **User Engagement**: Time spent on platform, return rate
- **NPS Score**: Net Promoter Score from user feedback

### Collaboration
- **Developer Handoff**: Smooth implementation with minimal design clarifications
- **Iteration Speed**: Rapid design updates based on feedback
- **Cross-Functional Alignment**: Product, design, and engineering alignment

## Sample Interview Questions for UI/UX Designer

1. **Portfolio Review**: Walk us through your design process for a complex web application.
2. **Case Study**: How would you design a file upload experience that builds user trust?
3. **Problem Solving**: A user says "I don't understand my ATS score." How would you redesign that screen?
4. **Accessibility**: How do you ensure your designs are accessible to users with disabilities?
5. **Collaboration**: Describe a time when a developer told you a design wasn't feasible. How did you handle it?
6. **User Research**: How would you validate whether users understand the gap analysis results?
7. **Design Systems**: Have you built a design system from scratch? What were the key components?
8. **Responsive Design**: How do you approach mobile-first vs desktop-first design?
